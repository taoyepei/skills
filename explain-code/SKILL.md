# explain-code

当用户输入 `/explain-code` 时，自动分析当前文件或选中的代码段，提供详细的代码解释。

## 使用方法

```
/explain-code [文件路径]
```

## 参数

- `file_path` (可选): 要解释的文件路径。如果不提供，则使用当前活动文件。

## 执行逻辑

当触发此 skill 时，执行以下步骤：

1. **确定目标文件**
   - 如果提供了 `file_path` 参数，使用该路径
   - 否则，使用当前活动编辑器中的文件

2. **检测文件类型**
   - 根据文件扩展名识别语言：
     - `.sv`, `.svh` → SystemVerilog
     - `.v`, `.vh` → Verilog
     - `.vhd`, `.vhdl` → VHDL
     - 其他扩展名按常规编程语言处理

3. **读取并分析代码**
   - 读取文件全部内容
   - 如果是代码段选择，只读取选中部分

4. **生成代码解释报告**，包含以下内容：

### 对于 SystemVerilog/UVM 代码：

- **整体功能概述**: 代码的主要目的和功能
- **模块/类说明**:
  - 模块名称和作用
  - 类继承关系（如果是 UVM）
  - 关键接口和端口
- **方法/任务描述**:
  - 函数/任务/always 块的功能
  - 敏感列表和触发条件
  - 关键算法逻辑
- **数据流分析**:
  - 信号流向
  - 状态机描述（如果有）
- **UVM 组件关系** (如果是 UVM 代码):
  - UVM 树结构
  - sequence/driver/monitor 关系
  - factory 注册和配置

### 对于其他编程语言：

- **功能概述**: 代码的主要功能
- **结构分析**: 类、函数、关键变量
- **关键逻辑**: 核心算法和流程
- **依赖关系**: 重要的导入和调用关系

5. **输出格式**
   - 使用中文输出解释
   - 使用 markdown 格式，包含适当的代码块和标题
   - 对于复杂的代码，使用表格或列表组织信息

## 示例

用户输入:
```
/explain-code
```

系统响应:
```
## 代码解释: my_uvm_test.sv

### 文件类型
SystemVerilog / UVM

### 整体功能概述
这是一个 UVM 测试用例，用于验证 DUT 的寄存器读写功能。

### 类结构
- **类名**: `my_uvm_test`
- **继承**: 继承自 `uvm_test`
- **功能**: 构建验证环境并启动测试序列

### 关键方法
1. **build_phase**
   - 功能: 实例化环境组件
   - 创建 `env` 和 `seq` 实例

2. **run_phase**
   - 功能: 执行测试序列
   - 启动 `reg_rw_sequence` 并等待完成

### UVM 组件关系
```
my_uvm_test
└── env (my_uvm_env)
    ├── agent
    │   ├── driver
    │   └── monitor
    └── scoreboard
```

### 数据流
- Sequence 产生 transaction → Driver 驱动 DUT
- Monitor 采样 DUT 输出 → Scoreboard 比对
```

## 注意事项

- 解释应简洁明了，避免过度详细的语法描述
- 重点关注代码的"为什么"和"做什么"，而非"怎么做"的细节
- 对于大型文件，先提供高层概述，再深入关键部分
