# Xprop配置规则

## 1. 必须包含的内容

### xprop模式说明
- 说明有哪些模式（tmerge, xn等）
- 解释各模式的含义

### 选择推荐模式的原因
- 解释为什么选择某个模式

### VCS编译选项示例
- 给出具体的编译命令

## 2. Xprop配置模板

```
阶段二：RTL Xprop Simulation

• Xprop 配置：开启 xprop 检测，检测 X 态传播
• Xprop 模式说明：
  - tmerge 模式：合并 X 态传播路径，当控制信号为 X 时，输出为 X 或其他可能值的合并结果，推荐用于验证
  - xn 模式：X 态直接传播到输出，适用于更严格的 X 态检测
  - 本项目采用 tmerge 模式，原因：既能检测 X 态问题，又能减少误报，适合功能验证
• VCS 编译选项示例：
  - vcs -xprop=tmerge -debug_access+all ...
• 目标：确保不存在 X 态导致的功能异常
```

## 3. 不要添加的内容

**禁止添加**：
- "Golden子模块处理"等模板中没有的内容
- 与模板无关的额外配置说明

## 4. 模式选择指南

| 模式 | 特点 | 适用场景 |
|---|---|---|
| tmerge | 合并X态路径，减少误报 | 功能验证（推荐） |
| xn | X态直接传播，严格检测 | 更严格的X态检测 |

## 5. 常见错误

| 错误 | 正确做法 |
|---|---|
| 只写"开启xprop检测" | 详细说明模式、原因、编译选项 |
| 添加"Golden子模块处理" | 不添加模板中没有的内容 |
| 没有编译选项示例 | 给出VCS编译选项示例 |
