# 覆盖率方案规则

## 1. 代码覆盖率（必须完整列出5种）

**必须包含**：

| 覆盖率类型 | 英文名称 | 说明 |
|---|---|---|
| 行覆盖率 | Line Coverage | 代码行执行覆盖 |
| 分支覆盖率 | Branch Coverage | if/else分支覆盖 |
| 跳转覆盖率 | Toggle Coverage | 信号翻转覆盖（容易遗漏！） |
| FSM覆盖率 | FSM Coverage | 状态机状态和转换覆盖 |
| 条件覆盖率 | Condition Coverage | 条件表达式覆盖 |

**目标值**：所有代码覆盖率目标100%

**示例**：
```
代码覆盖率：
• 行覆盖率（Line Coverage）：目标 100%
• 分支覆盖率（Branch Coverage）：目标 100%
• 跳转覆盖率（Toggle Coverage）：目标 100%
• FSM覆盖率：目标 100%
• 条件覆盖率（Condition Coverage）：目标 100%
```

## 2. 功能覆盖率

### 分bin方案
- 列出需要覆盖的功能点
- 为每个功能点定义bins

### Cross Coverage方案
- 说明哪些功能点需要交叉覆盖
- 解释为什么需要交叉覆盖

### 采样方案
- 说明何时采样（如每个transaction结束时）
- 说明如何采样

## 3. 功能覆盖点示例

根据LRS中的功能确定覆盖点：

```
一、功能覆盖点

命令类型覆盖：
• bins: {列出LRS中定义的所有命令类型}
• 目标：100% 覆盖所有命令类型

操作模式覆盖：
• bins: {列出LRS中定义的所有模式}
• 目标：100% 覆盖所有模式

错误码覆盖：
• bins: {列出LRS中定义的所有错误类型}
• 目标：100% 覆盖所有错误场景

二、Cross Coverage：
• 命令类型 × 操作模式：确保每种命令在各模式下都被测试
• 命令类型 × 错误码：确保每种命令的错误场景覆盖
```

## 4. 采样方案模板

```
采样方案：
• 功能覆盖使用 covergroup 在 transaction 级别采样
• 代码覆盖使用 VCS 自带工具收集
• 在每个测试用例结束时合并覆盖率数据
```

## 5. 常见错误

| 错误 | 正确做法 |
|---|---|
| 漏掉Toggle Coverage | 必须列出全部5种代码覆盖率 |
| 未说明目标值 | 所有覆盖率目标100% |
| 功能覆盖点不完整 | 根据LRS功能完整列出 |
