# 常见错误清单

生成验证方案文档后，逐项检查以下内容：

## 错误清单

| 序号 | 检查项 | 常见错误 | 正确做法 |
|:---:|--------|---------|---------|
| 1 | 模板固定内容 | 在3.2节等固定内容处添加额外内容 | 保持固定内容不变，其他内容放到正确章节 |
| 2 | Agent显示 | 多个Agent放在同一个"Agents"框内 | 每个Agent独立显示 |
| 3 | Interface标注 | 连线上没有标注Interface名称 | 在连线上用 `\|Interface名\|` 标注 |
| 4 | 双向箭头 | 用单向箭头表示双向交互 | 用 `<===>` 表示双向交互 |
| 5 | CRG连接 | Agent通过CRG连接到DUT | Agent直接通过Interface连接DUT |
| 6 | Scoreboard连线 | Scoreboard的输入连线未画完整 | 完整画出Agent→Scoreboard和Ref→Scoreboard |
| 7 | Coverage Collector数据来源 | 只写从一个Agent收集 | 列出所有Agent作为数据来源 |
| 8 | 组件描述 | 描述未在数据流图中出现的组件（如CSR Agent） | 只描述数据流中出现的组件 |
| 9 | Monitor实现方案 | 缺少协议违规检测描述 | Monitor必须包含协议违规检测 |
| 10 | Checker表述 | 用"监控事务"代替"比对事务" | Monitor监控，Checker比对 |
| 11 | 代码覆盖率 | 漏掉Toggle Coverage（跳转覆盖率） | 必须列出全部5种代码覆盖率 |
| 12 | Xprop配置 | 只写"开启xprop检测" | 详细说明模式、原因、编译选项 |
| 13 | Xprop额外内容 | 添加"Golden子模块处理"等模板没有的内容 | 不添加模板中没有的内容 |
| 14 | Mermaid流程图 | 存在孤立节点（没有输入或输出的节点） | 确保所有节点正确连接 |
| 15 | 内容来源 | 添加LRS中没有的架构信息（如内部架构组成表格） | 所有内容来源于LRS |
| 16 | 测试用例分类 | 在6.1节添加测试用例分类（如用户不需要） | 根据用户需求决定是否保留 |
| 17 | 比对内容描述 | Checker用"监控总线事务"表述 | 用"比对事务的地址、数据正确性" |
| 18 | 参考文档 | 添加用户未提供的参考文档（如"SoC集成规范"等） | 只列出用户提供的LRS文档 |
| 19 | 架构图 | 2.1节没有绘制mermaid架构图 | 绘制模块位置和连接关系的mermaid图 |
| 20 | 验证策略 | 验证难点只列出问题，没有写应对策略 | 每个难点后写"应对策略：..." |
| 21 | Checker内容 | 包含"状态码比对"等应在testcase中校验的内容 | 只保留需要Scoreboard比对的内容 |
| 22 | 环境重用方案 | 只写"采用标准架构"、"便于重用"等笼统描述 | 详细描述每个组件的重用策略 |
| 23 | Transaction定义 | Transaction字段位宽与LRS不一致 | 严格按照LRS文档定义位宽 |

## 检查方式

生成文档后，按以下步骤检查：

### Step 1: 检查模板固定内容
- [ ] 3.2节是否只有一行固定内容？
- [ ] 是否有不属于该章节的内容？

### Step 2: 检查验证环境框图
- [ ] Agent是否各自独立显示？
- [ ] 连线上是否标注了Interface？
- [ ] 双向交互是否用双向箭头？
- [ ] Scoreboard连线是否完整？
- [ ] Coverage Collector是否连接所有Agent？

### Step 3: 检查数据流描述
- [ ] 各组件输入输出是否明确？
- [ ] Scoreboard比对哪两路数据是否说明？
- [ ] Coverage Collector数据来源是否完整列出？

### Step 4: 检查Agent实现方案
- [ ] 每个Agent是否都有Interface/Transaction/Driver/Monitor？
- [ ] Monitor是否包含协议违规检测？
- [ ] 是否有未在数据流中的Agent被描述？

### Step 5: 检查Checker实现方案
- [ ] 是否用"比对"而非"监控"？
- [ ] 数据来源是否明确？

### Step 6: 检查覆盖率方案
- [ ] 代码覆盖率是否包含全部5种？
- [ ] Toggle Coverage是否遗漏？

### Step 7: 检查Xprop配置
- [ ] 是否说明了模式？
- [ ] 是否说明了选择原因？
- [ ] 是否给出了编译选项？
- [ ] 是否添加了无关内容？

### Step 8: 检查Mermaid流程图
- [ ] 是否有孤立节点？
- [ ] 流程是否完整？

### Step 9: 检查内容来源
- [ ] 所有内容是否来源于LRS？
- [ ] 是否添加了LRS中没有的信息？

### Step 10: 检查参考文档
- [ ] 参考文档是否只包含用户提供的LRS？
- [ ] 是否添加了"SoC集成规范"等未提供的文档？

### Step 11: 检查架构图
- [ ] 2.1节是否包含mermaid架构图？
- [ ] 架构图是否体现模块位置和连接关系？

### Step 12: 检查验证策略
- [ ] 每个验证难点是否有对应的应对策略？
- [ ] 应对策略是否具体可执行？

### Step 13: 检查Checker比对内容
- [ ] 比对内容是否只包含需要Scoreboard比对的数据？
- [ ] 是否包含了"状态码比对"等应在testcase中校验的内容？

### Step 14: 检查环境重用方案
- [ ] 是否详细描述了每个组件的重用策略？
- [ ] 是否说明了如何被更高层次验证平台重用？

### Step 15: 检查Transaction定义
- [ ] Transaction字段位宽是否与LRS一致？
- [ ] 是否有凭空添加的字段？
